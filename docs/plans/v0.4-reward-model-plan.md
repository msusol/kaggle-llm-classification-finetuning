# v0.4 — Reward Model Zero-Shot Classifier Plan

## Goal

Use a purpose-built reward model as a **zero-shot classifier** for human preference
prediction, bypassing the SFT training step entirely. Reward models are pre-trained to
score (prompt, response) pairs by human preference — the exact task this competition measures.

## Context

Standard SFT approach (v0.2, v0.3): fine-tune a generative LLM to output A/B/tie as the
next token. Requires training time and a GPU/TPU run slot per fold.

Reward model approach: load a pre-trained reward model, score response_a and response_b
independently, convert the scalar delta to 3-class probabilities. Zero training required.

The gap to bridge: reward models have no native tie output. Tie probability must be
synthesized from score proximity and calibrated on an OOF fold.

## Inference approach

```python
# Score each response independently
score_a = reward_model(tokenize(prompt + response_a))  # scalar logit
score_b = reward_model(tokenize(prompt + response_b))  # scalar logit

# Sigmoid over delta → A/B preference probabilities
diff  = score_a - score_b
p_a   = sigmoid(diff * temperature)
p_b   = sigmoid(-diff * temperature)

# Tie: inverse of preference confidence; renormalize
raw_tie = (1 - abs(p_a - p_b)) * tie_weight
total   = p_a + p_b + raw_tie
probs   = [p_a / total, p_b / total, raw_tie / total]
```

Two hyperparameters to tune on OOF fold 0:
- `temperature`: sharpens/softens the sigmoid (start at 1.0)
- `tie_weight`: scales tie mass relative to A/B (start at 0.3)

## Swap-TTA

Apply the same swap augmentation as SFT notebooks:
- Forward: score (prompt, response_a) vs (prompt, response_b)
- Swapped: score (prompt, response_b) vs (prompt, response_a), then flip A/B indices
- Average the two probability vectors

## v0.4c — classification head (optional)

If zero-shot calibration is insufficient, freeze the reward backbone and train a
3-class linear head on the pooled reward score features:

```python
# Replace reward head with 3-class head
class RewardClassifier(nn.Module):
    def __init__(self, backbone):
        super().__init__()
        self.backbone = backbone          # frozen
        self.head     = nn.Linear(hidden, 3)

    def forward(self, input_ids, attention_mask):
        h = self.backbone.model(input_ids, attention_mask).last_hidden_state[:, -1]
        return self.head(h)
```

Train head only (frozen backbone) on fold train split with cross-entropy.
Runtime: fast — only the head has gradients; can run on T4 in < 1 hour.
