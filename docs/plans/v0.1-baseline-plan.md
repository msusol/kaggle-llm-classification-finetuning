# v0.1 — Baseline Plan (rungs 0–1)

Goal: lock the offline submission pipeline and a CV harness with a cheap CPU model.

## Steps

1. `scripts/download_data.sh` → `data/train.csv`, `data/test.csv`, `data/sample_submission.csv`.
2. EDA notebook (`notebook/eda.ipynb`):
   - class balance of `winner_model_a/b/tie`
   - response length distributions; length-diff vs. winner
   - position-bias check: does response A win more often?
   - missing/empty responses, multi-turn prompt frequency
3. Feature build (`scripts/build_features.py`):
   - TF-IDF (word + char n-grams) on `response_a`, `response_b`, `prompt`
   - len(a), len(b), len_ratio, len_diff
   - token overlap(prompt, a/b), response==empty flags
4. Model: LightGBM multiclass (`num_class=3`, `objective=multiclass`), 5-fold stratified.
5. Predict test, write `submission.csv`, verify header + row count.

## Acceptance

- CV log loss < 1.0986 (beats uniform).
- Offline notebook produces a valid `submission.csv`.
- CV → LB gap recorded in [leaderboard.md](leaderboard.md).

## Risks

- TF-IDF alone barely beats uniform; treat this purely as plumbing validation, not a
  competitive score. The real gains start at v0.2.
