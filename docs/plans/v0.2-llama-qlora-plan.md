# v0.2 — Llama-3.2-3B LoRA (pipeline validation)

## Goal

Validate the end-to-end SFT pipeline on Kaggle using `meta-llama/Llama-3.2-3B-Instruct`
with LoRA. This is a **pipeline validation run only** — the model is too small to be
competitive. Success = notebook runs, OOF score is recorded, submission.csv is valid.

## Status

**Abandoned.** Three attempts to fix the Kaggle TPU 2-hour idle timeout all failed
(v30/v32/v34/v35). DGX estimated ~100h to complete. v0.2 is unviable on both platforms.

See `docs/investigate/2026-06-27-v02-t4x2-ddp.md` §2 for the full idle-timeout history.
Proceed directly to v0.4 (reward model zero-shot on T4 — no training required).

## Approach

- Llama-3.2-3B-Instruct with LoRA (r=16)
- Label-token readout: softmax over A/B/tie logits at final token position
- Swap augmentation in training + swap-TTA at inference
- 5-fold stratified split; fold 0 first

## Hardware journey

| Version | Hardware | Outcome |
|---|---|---|
| v24 | Kaggle T4 (single, `CUDA_VISIBLE_DEVICES=0`) | Ran 11+ hours; stopped before completing |
| v25 | Kaggle T4×2 (DDP attempt) | Failed — DataParallel + TRL incompatible in notebook |
| v26–v28 | Kaggle TPU v5e-8 (TPU migration) | Auto-run errors fixed with guard assertion |
| v29 | Kaggle TPU v5e-8 | Cancelled — 2h TPU idle timeout (single-threaded tokenization of 91K examples) |
| v30/v32 | Kaggle TPU v5e-8 | `num_proc=4` + 50K sample cap → still hit 2h idle (multiprocessing likely restricted in Kaggle) |
| v34/v35 | Kaggle TPU v5e-8 | Deferred `xm.xla_device()` + 10K samples → still hit 2h idle (`import torch_xla` itself likely allocates the TPU) |
| — | DGX Spark GB10 | Estimated ~100h to complete — not viable |

T4×2 with SFTTrainer is a dead end; TPU v5e-8 idle timeout is unsolvable from notebook context.
v0.2 is abandoned.

## Key TPU adaptations

| Setting | Value | Reason |
|---|---|---|
| `torch_dtype` | `bfloat16` | fp16 unsupported by XLA |
| `attn_implementation` | `'eager'` | Flash/SDPA not supported on TPU XLA |
| `bf16` | `True` | XLA-native precision |
| `fp16` | `False` | |
| `dataloader_drop_last` | `True` | XLA requires static batch shapes |
| `device_map` | not set | XLA handles placement automatically |
| `ddp_find_unused_parameters` | not set | Not applicable on TPU |

## Notebook

`notebook/v0.2-llama-qlora.ipynb` + `notebook/v0.2-llama-qlora-kernel-metadata.json`

fail fast and clearly if run on the wrong accelerator.

## Exit criteria

- Fold 0 completes without OOM or XLA error
- OOF log loss recorded in `leaderboard.md`
- `submission.csv` generated with valid probabilities
- Proceed to v0.3 (Gemma-2-9b) regardless of score — this is pipeline validation only
