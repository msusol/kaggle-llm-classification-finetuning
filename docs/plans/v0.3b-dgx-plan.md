# v0.3b — Gemma-2-9b DGX Spark GB10 Plan

## Goal

Train `gemma-2-9b-it` with LoRA across all 5 folds in bf16 on the DGX Spark GB10
(120 GB VRAM), average the fold adapters, upload to Kaggle, and submit via a separate
offline inference notebook.

This is the production version of v0.3. v0.3 (TPU) validates the pipeline on fold 0;
v0.3b runs all 5 folds with more VRAM headroom and no XLA constraints.

## Why DGX over TPU for the full run

| | Kaggle TPU v5e-8 | DGX Spark GB10 |
|---|---|---|
| VRAM | 8 × 16 GB = 128 GB | 120 GB (single socket) |
| Folds | 1 per run (9 h limit) | All 5 in one session |
| dtype | bf16 (XLA only) | bf16 (native CUDA) |
| Quantization | None needed | None needed |
| Session limit | 9 h / 30 h/week quota | Unlimited (local) |
| Iteration speed | Slow (queue wait + UI steps) | Immediate |

## Prerequisites

- v0.3 TPU fold 0 OOF < 0.95 (pipeline validated)
- DGX has `google/gemma-2-9b-it` weights available (HuggingFace cache or local copy)
- Reuse LoRA/PEFT stack from `mineral-hr-llm` harness (already validated on DGX)

## Training approach

Same hyperparameters as v0.3 but no XLA constraints:

| Param | Value |
|---|---|
| dtype | bf16 |
| `attn_implementation` | `sdpa` (flash attention available on DGX) |
| `MAX_SEQ_LEN` | 1536 |
| `BATCH_SIZE` | 4–8 (tune to fill VRAM) |
| `GRAD_ACCUM` | 2–4 |
| `LORA_R` | 16 |
| Folds | All 5 |

Run under `tmux` on `sparkdb62`. Never background without tmux — broken pipe kills the job.

## LoRA averaging

After all 5 folds complete, merge adapters by averaging LoRA A/B matrices element-wise:

```python
# scripts/average_lora.py (to be created)
import torch
from peft import PeftModel
from transformers import AutoModelForCausalLM

adapter_dirs = [f'/path/to/fold{i}-adapter' for i in range(5)]
# Load first adapter as base
model = AutoModelForCausalLM.from_pretrained(BASE_MODEL, torch_dtype=torch.bfloat16)
model = PeftModel.from_pretrained(model, adapter_dirs[0])
state = dict(model.named_parameters())

# Average remaining adapters into state
for d in adapter_dirs[1:]:
    m2 = PeftModel.from_pretrained(
        AutoModelForCausalLM.from_pretrained(BASE_MODEL, torch_dtype=torch.bfloat16), d
    )
    for k, v in m2.named_parameters():
        if 'lora' in k:
            state[k].data += v.data
for k in state:
    if 'lora' in k:
        state[k].data /= len(adapter_dirs)

model.save_pretrained('/path/to/averaged-adapter')
```

Output: single merged adapter (~200–300 MB).
