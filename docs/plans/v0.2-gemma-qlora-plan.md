# v0.2 — Gemma-2-9b QLoRA Plan (rung 3)

Train `gemma-2-9b-it` as a 3-class sequence classifier with QLoRA on the DGX Spark
(GB10, `sparkdb62`), reusing the working LoRA/PEFT harness from **`mineral-hr-llm`**.

## Why this model

`gemma-2-9b-it` was the consensus backbone in top LMSYS solutions, outperforming
Llama-3/3.1-8b at this task.[cite:3][cite:4] Sequence-classification finetuning beats
prompting an off-the-shelf judge, which suffers position/verbosity/self-enhancement
bias.[cite:1][cite:2]

## Reuse from `mineral-hr-llm` (sparkdb62)

That project already has a working LoRA PEFT training loop on the Spark. Fork its
harness and change:
- **Task head:** swap causal-LM / regression head for `Gemma2ForSequenceClassification`
  with `num_labels=3`.
- **Data collator:** build inputs from `prompt + response_a + response_b`; labels are
  argmax of the three winner columns (0=a, 1=b, 2=tie).
- **Loss:** cross-entropy on the 3-class head (HF `Trainer` default for classification).
- Keep its bf16 + 4-bit NF4 bitsandbytes config, tmux launch pattern, and checkpointing.

## Input construction

```
<bos>USER PROMPT:
{prompt}

RESPONSE A:
{response_a}

RESPONSE B:
{response_b}
```

- Tokenize with a max length cap (start 1536–2048; raise if runtime allows).
- On overflow, keep the **tail** of each response.
- **A/B swap augmentation:** for a fraction of rows, swap response_a/response_b and
  swap labels 0↔1 (tie unchanged) to fight position bias.

## Training config (starting point)

| Param | Value |
|---|---|
| LoRA r / alpha | 16–32 / 32–64 |
| Target modules | q,k,v,o,gate,up,down proj |
| Quant | 4-bit NF4, double-quant, bf16 compute |
| LR | 1e-4 → 2e-4 (cosine, warmup 3%) |
| Epochs | 1–2 (watch overfit) |
| Batch / grad-accum | tuned to GB10 HBM |
| Folds | 5-fold stratified |

## CV & selection

- 5-fold; report per-fold log loss + OOF log loss.
- Select checkpoints by **OOF log loss**, not accuracy.
- Log every run in [leaderboard.md](leaderboard.md).

## Offline submission packaging

1. Upload `gemma-2-9b-it` base as a Kaggle model/dataset input.
2. Upload the trained adapter (`adapter_config.json` + weights) as a private dataset.
3. Upload any required wheels (bitsandbytes, etc.).
4. Inference notebook: load base + adapter offline, **swap-TTA** (a,b)+(b,a) averaged,
   batch over ~25k rows within 9 h, softmax → `submission.csv`.

## Acceptance

- OOF log loss in the ~0.89–0.92 range.
- Inference notebook completes < 9 h on the full hidden test (verify on example set first).
- Run through [submission-checklist.md](submission-checklist.md) before submitting.
